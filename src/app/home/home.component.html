Welcome To RunDash

Purpose

Somewhere in early May 2019, I decided that I wanted to run a marathon. Even though I was a complete beginner, there was a lot of work to be done in terms of training. I run every morning to start my day, but I haven't set goals. My usual distance was about 2 or 3 miles everday; It was just that. There was no growth. I needed to progress.

There are a lot of variables such as soreness, fatigue, and injury that can affect the quality of a run. And if you are pushing yourself, its even more difficult to pinpoint on how to react. Should I rest? Should I eat more? Is my form right? Should I longer? Faster?

I've tried running apps such as Strava, RunKeeper, Apple Fitness, and Nike Run Club. They are all great in terms of keep tracking of your runs but I needed to extract more value from the data. I needed to find another way to take control of my own data.

I decided to create my own web application. I wanted the data in my own hands. I wanted to see if there ways to strategically implement tactics to reach my goal of running a marathon.

Technologies

The web application uses Django Rest Framework to serve my own Apple Health Data. The front-end is created with Angular, Material, and Ng-charts for data visualization.

Note: This is an ongoing project ... features and changes to the websites are likely as I get more ideas
